# 07. 집필 원칙 (연결성/정합성 필수 규칙)

## 1) 멀티 시점 연결 원칙
- 5인 서사는 분리된 단편이 아니라 하나의 메인 타임라인을 공유한다.
- 각 화의 사건은 다른 인물 화에서 반드시 이어지거나 반영되어야 한다.
- 어떤 인물 화에서도 "세계관 밖 독립 사건"을 만들지 않는다.

## 2) 대화/사건 정합성 원칙
- 캐릭터들이 같은 시간, 같은 장소에서 만난 장면의 대화는 5인 화에서 동일해야 한다.
- 시점 차이로 추가되는 정보는 허용하되, 기존 대사를 바꾸거나 의미를 뒤집지 않는다.
- 시간, 장소, 참석자, 핵심 발언은 모든 시점에서 일치해야 한다.

## 3) 집필 전 필수 검토 원칙
- 새 화를 쓰기 전 아래 자료를 모두 확인한다.
1. `webnovel-innugroup/01_worldbuilding.md`
2. `webnovel-innugroup/02_characters.md`
3. `webnovel-innugroup/03_timeline_2010_2026.md`
4. `webnovel-innugroup/04_release_structure.md`
5. `webnovel-innugroup/05_season1_episode_blueprint.md`
6. `webnovel-innugroup/06_org_structure_2010.md`
7. `webnovel-innugroup/characters-by-year/해당연도.md`
8. 기존 작성된 모든 화(5인 전원 분량 포함)

## 4) 집필 순서 고정 원칙
- 한 주차 집필 순서는 고정한다.
1. 강서준
2. 한도희
3. 윤재훈
4. 문하린
5. 오민석

## 5) 충돌 발생 시 우선순위
- 설정 충돌이 발생하면 아래 우선순위를 따른다.
1. 연표/세계관 고정값 (01, 03)
2. 인물 바이블/연도별 인물 상태 (02, characters-by-year)
3. 기존 공개 화의 확정 대사/사건
4. 신규 초안

## 6) 회차 작성 체크리스트 (매 화 적용)
- [ ] 해당 주차의 공통 사건을 정의했는가
- [ ] 같은 만남 장면의 핵심 대사가 기존 화와 일치하는가
- [ ] 다른 4인 화와 시간축이 모순되지 않는가
- [ ] 직책/호칭/관계 온도가 해당 연도 설정과 일치하는가
- [ ] 다음 화로 넘길 떡밥이 설정 충돌 없이 연결되는가

## 7) 설정 추가/변경 관리 원칙
- 글 진행 중 새로운 캐릭터, 사건, 조직 설정이 추가되면 문서를 즉시 갱신한다.
- 반영 방식은 둘 중 하나를 선택하되, 누락 없이 기록한다.
1. 기존 문서 업데이트 (02, 03, characters-by-year 등)
2. 신규 보조 문서 생성 후 `README.md` 파일 목록에 등록
- 모든 변경 사항은 `08_story_update_log.md`에 날짜/변경 요약/영향 범위를 남긴다.

## 8) 작명/고유명사 원칙
- 대학, 기업, 기관, 언론, 지명 기반 조직명 등은 모두 가상 이름을 사용한다.
- 실존 학교/기업/기관명은 본문과 설정 문서에 직접 사용하지 않는다.
- 새 고유명사를 만들면 `02_characters.md` 또는 관련 문서에 출처/의미를 함께 기록한다.
